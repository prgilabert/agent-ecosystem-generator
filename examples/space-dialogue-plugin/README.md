# space-dialogue-plugin

Generates short dramatic-comedy dialogue stories between a fixed astronaut and a robot assistant from a user prompt, saved as a Markdown file. Use when the user invokes `/escribe-dialogo` or asks to write a space dialogue, relato, or historia with two characters.

**Origin brief** (in Spanish):
> "quiero que me generes diálogos sobre prompts de situaciones en el espacio con un astronauta y un robot. Quiero una comedia dramática."

## Pattern

**orchestrator-workers** — plot-writer runs first to produce a shared outline; astronaut-voice and robot-voice then run **in parallel** consuming that outline; dialogue-reviewer assembles, polishes, and saves. Three sequential phases with a parallel fan-out in the middle.

## Primitives

- Orchestrator: `/escribe-dialogo` — see [`commands/escribe-dialogo.md`](./commands/escribe-dialogo.md)
- Sub-agents: `plot-writer`, `astronaut-voice`, `robot-voice`, `dialogue-reviewer` (see [`agents/`](./agents))
- Skills: (none — the pattern didn't need one)
- MCPs: (none — only local filesystem via Write/Bash)

## Usage

```
/escribe-dialogo una tormenta de meteoros inesperada cerca de Júpiter
```

Produces `dialogue-<slug>-<timestamp>.md` in the current working directory.

## What this example demonstrates

- **Mixed sequential + parallel** orchestration (not pure fan-out).
- **Model tiering:** sonnet for plot-writer/astronaut-voice/dialogue-reviewer, haiku for robot-voice (cheaper model justified because the voice is constrained).
- **Bilingual triggers:** the primary trigger phrases include both Spanish ("crea un dialogo sobre", "escribe una historia de") and English, so the same command fires across languages.
- **Scoring 0.95** on the validator — one warning about `coverage` because the spec's sub_tasks were generic verbs ("generate-dialogue", "apply-tone") that don't map cleanly to individual agents; the pattern is still correct.
